# 桌面端响应式设计规范

## 1. 项目说明

本网站为 HMI / UX Designer 个人作品集网站，主要用于设计面试、作品展示和发送给面试官浏览。

网站以桌面端体验为主，Figma 设计稿基准宽度为 **1440px**。

响应式设计需要在保证设计稿视觉还原度的前提下，使页面在不同桌面屏幕宽度下保持稳定、清晰和一致。

---

## 2. 核心响应式尺寸

- 最小内容宽度：`1028px`
- 设计基准宽度：`1440px`
- 最大内容宽度：`1920px`

页面主要在 `1028px–1920px` 范围内进行响应式变化。

---

## 3. 1440px 基准规则

当浏览器视口宽度为 `1440px` 时，必须严格按照 Figma 设计稿进行还原。

需要保持：

- 页面布局
- 栅格结构
- 内容宽度
- 图片尺寸
- 视频尺寸
- 组件尺寸
- 字体字号
- 字体字重
- 行高
- 模块间距
- 页面内边距
- 圆角
- 描边
- 阴影
- 图文比例
- 内容层级关系

`1440px` 是所有其他桌面宽度进行响应式计算的基准。

---

## 4. 1028px–1920px 流式响应区间

在浏览器视口宽度为 `1028px–1920px` 时，页面采用流式桌面端响应式布局。

必须遵守以下规则：

- 保持 Figma 设计稿中的页面结构。
- 保持内容顺序不变。
- 保持模块层级关系不变。
- 保持栅格和对齐关系。
- 图片、视频、卡片和主要视觉组件根据可用宽度自适应缩放。
- 所有媒体资源必须保持原始宽高比。
- 禁止图片、视频和组件被拉伸变形。
- 字号、组件尺寸和间距采用受控缩放。
- 所有受控尺寸必须设置合理的最小值和最大值。
- 不允许使用 `transform: scale()` 对整个页面进行整体缩放。
- 不允许通过 JavaScript 强制缩放整个网站。
- 不因屏幕宽度变化随意修改列数、模块结构或内容顺序。

优先使用：

- CSS Grid
- Flexbox
- 百分比宽度
- `clamp()`
- `min()`
- `max()`
- `min-width`
- `max-width`
- `aspect-ratio`

---

## 5. 1440px–1920px 放大规则

当浏览器宽度大于 `1440px` 且小于等于 `1920px` 时，页面可以适度放大。

具体要求：

- 图片、视频和大型视觉模块可以根据内容区宽度放大。
- 卡片、背景模块和视觉容器可以按比例增加尺寸。
- 主标题可以轻微放大。
- 正文字号只允许小幅放大。
- 导航、按钮、标签和图标不能无限同比放大。
- 页面左右内边距可以随着浏览器宽度增加。
- 模块之间的纵向间距可以保持原设计或轻微增加。
- 页面整体应保持舒展，但不能显得空散。
- 所有内容在 `1920px` 时达到最大尺寸。

建议使用受控尺寸，例如：

```css
.hero-title {
  font-size: clamp(56px, 4.44vw, 72px);
}

.page-container {
  padding-inline: clamp(80px, 5.5vw, 120px);
}
```

---

## 6. 大于或等于 1920px 的规则

当浏览器视口宽度大于或等于 `1920px` 时：

- 网站核心内容不再继续放大。
- 内容区最大宽度固定为 `1920px`。
- 图片、文字、组件和模块保持 `1920px` 状态下的最大尺寸。
- 内容区域水平居中。
- 浏览器继续变宽时，只增加页面两侧的外部留白。
- 页面背景允许继续铺满整个浏览器视口。
- 不允许继续扩大字号、图片、卡片或组件尺寸。

推荐结构：

```css
.page-shell {
  width: 100%;
}

.page-container {
  width: 100%;
  max-width: 1920px;
  margin-inline: auto;
}
```

---

## 7. 1028px–1440px 缩小规则

当浏览器宽度大于等于 `1028px` 且小于 `1440px` 时：

- 保持桌面端布局。
- 不切换为移动端布局。
- 不改变主要模块的排列结构。
- 图片、视频和大型卡片按比例缩小。
- 页面内容区随浏览器宽度缩小。
- 文字字号采用受控缩小。
- 导航、按钮和标签必须保持清晰可读。
- 点击区域不得缩小到影响操作。
- 页面内边距和模块间距可以适度缩小。
- 间距不能完全固定，也不能无限压缩。
- 页面不能因为缩小而产生文字重叠、图片溢出或组件挤压。

建议使用受控字号，例如：

```css
.hero-title {
  font-size: clamp(48px, 4.44vw, 64px);
}

.body-text {
  font-size: clamp(16px, 1.25vw, 18px);
}
```

---

## 8. 小于或等于 1028px 的规则

当浏览器视口宽度小于或等于 `1028px` 时：

- 页面最小内容宽度固定为 `1028px`。
- 图片、文字、组件和模块不再继续缩小。
- 不自动切换为移动端布局。
- 不重新排列作品集内容。
- 不将桌面多列结构强制改为单列。
- 不压缩作品图片。
- 不缩小到影响文字可读性。
- 浏览器视口小于 `1028px` 时，允许出现横向滚动。

推荐实现：

```css
html,
body {
  min-width: 1028px;
}

.page-container {
  min-width: 1028px;
}
```

该规则适用于本网站以电脑端面试展示为主要使用场景的定位。

---

## 9. 图片、视频和媒体资源规则

所有图片、视频和媒体资源必须：

- 使用响应式宽度。
- 保持原始宽高比。
- 禁止拉伸。
- 禁止压扁。
- 禁止因容器变化而失真。
- 根据设计稿决定使用完整显示或裁切显示。
- 如需裁切，必须使用明确的 `object-fit` 规则。
- 不允许默认使用不受控制的 `cover` 导致重要内容被裁切。

推荐：

```css
.media {
  width: 100%;
  height: auto;
  display: block;
}

.cover-media {
  width: 100%;
  height: 100%;
  object-fit: cover;
}

.contain-media {
  width: 100%;
  height: 100%;
  object-fit: contain;
}
```

如 Figma 设计稿中存在固定比例容器，应使用：

```css
.media-wrapper {
  aspect-ratio: 16 / 9;
}
```

---

## 10. 文字响应式规则

文字不能与页面整体完全同比例缩放。

必须使用受控缩放策略：

- 大标题可以适度缩放。
- 小标题可以轻微缩放。
- 正文字号只允许在有限范围内变化。
- 导航、按钮、标签和说明文字优先保证可读性。
- 正文字号不能小于设计规范中的最低可读值。
- 行高需要随字号保持合理比例。
- 不允许因缩放导致文字换行严重变化。
- 不允许因缩放产生溢出、截断或重叠。

推荐使用：

```css
.title {
  font-size: clamp(48px, 4.44vw, 64px);
}

.subtitle {
  font-size: clamp(24px, 2.22vw, 32px);
}

.body {
  font-size: clamp(16px, 1.25vw, 18px);
}
```

具体数值应以 `notes/typography-rules.md` 为准。

---

## 11. 间距响应式规则

内容之间的间距不建议完全固定。

应使用受控变化规则：

- 保持 Figma 设计稿中的视觉节奏。
- 保持模块之间的层级关系。
- 大模块间距可以适度缩放。
- 小组件内部间距应尽量稳定。
- 按钮内部 Padding 不应随页面大幅缩放。
- 文字与图标之间的间距不应无限放大。
- 模块间距必须设置最小值和最大值。
- 在 `1028px` 时不能因为间距过大导致内容拥挤。
- 在 `1920px` 时不能因为间距过小导致页面显得局促。

推荐：

```css
.section {
  padding-block: clamp(80px, 8vw, 120px);
}

.content-grid {
  gap: clamp(24px, 2.78vw, 40px);
}

.card {
  padding: clamp(24px, 2.5vw, 36px);
}
```

---

## 12. 组件响应式规则

组件需要根据类型选择不同的响应策略。

### 大型视觉组件

例如：

- 项目封面
- Hero Banner
- 项目图片
- 视频展示区域
- 大型数据卡片

可以根据内容区宽度明显缩放。

### 功能型组件

例如：

- 导航栏
- 按钮
- 标签
- Tab
- 图标
- 分页器
- 返回按钮

只允许轻微缩放或保持稳定尺寸。

### 文本组件

例如：

- 项目标题
- 项目描述
- 说明文字
- 数据标签

必须优先保证阅读体验，不能单纯等比缩放。

### 组件内部规则

- 内部对齐关系不变。
- 图标与文字比例不变。
- 圆角保持统一设计语言。
- 描边粗细不能因屏幕变化过度放大或缩小。
- Hover、Active、Selected 等状态尺寸必须保持一致。
- 不允许因响应式变化导致组件跳动。

---

## 13. 栅格与布局规则

- 1440px 下严格还原 Figma 栅格。
- 1028px–1920px 范围内允许列宽流式变化。
- 列间距采用受控缩放。
- 主要内容必须保持对齐。
- 不允许出现随机宽度。
- 不允许通过大量绝对定位实现主要布局。
- 绝对定位仅用于明确的装饰性元素。
- 页面主要结构必须使用 CSS Grid 或 Flexbox。
- 不允许为了适配而随意隐藏内容。
- 不允许改变作品展示顺序。

---

## 14. 页面容器建议

推荐页面结构：

```html
<body>
  <div class="page-shell">
    <main class="page-container">
      <!-- 页面内容 -->
    </main>
  </div>
</body>
```

推荐样式：

```css
html,
body {
  margin: 0;
  min-width: 1028px;
}

.page-shell {
  width: 100%;
}

.page-container {
  width: 100%;
  min-width: 1028px;
  max-width: 1920px;
  margin-inline: auto;
}
```

实际左右 Padding、Grid 和内容最大宽度应结合 Figma 设计稿进行设置。

---

## 15. 禁止事项

禁止以下实现方式：

- 使用 `transform: scale()` 缩放整个页面。
- 使用固定缩放倍数适配不同屏幕。
- 使用 JavaScript 强制计算整页缩放比例。
- 使用绝对定位搭建主要页面结构。
- 所有文字、图标和按钮完全同比例放大缩小。
- 所有间距完全固定不变。
- 所有间距无限同比例缩放。
- 图片被拉伸或压缩变形。
- 在 1028px 以下自动改成移动端布局。
- 在 1920px 以上继续放大核心内容。
- 随意改变 Figma 设计稿中的模块顺序。
- 为了适配屏幕隐藏重要作品内容。

---

## 16. Codex 执行要求

Codex 开发时必须：

1. 以 `1440px` Figma 设计稿为还原基准。
2. 使用 `1028px` 作为页面最小内容宽度。
3. 使用 `1920px` 作为页面最大内容宽度。
4. 在 `1028px–1920px` 范围内实现流式桌面端响应式。
5. 图片、视频和大型视觉模块根据宽度自适应缩放。
6. 文字、按钮、图标和功能组件采用受控缩放。
7. 模块间距和页面 Padding 使用 `clamp()` 等方式控制最小值和最大值。
8. 确保所有图片、视频和组件比例稳定。
9. 确保文字在所有桌面宽度下清晰可读。
10. 确保页面在面试官常用的笔记本和桌面显示器上均可正常浏览。
11. 完成后分别检查以下宽度：
    - `1028px`
    - `1280px`
    - `1440px`
    - `1600px`
    - `1920px`
    - `2560px`
12. 检查是否存在：
    - 横向溢出
    - 图片变形
    - 文字重叠
    - 内容截断
    - 组件错位
    - 间距异常
    - 字号过大或过小

---

## 17. 验收标准

响应式开发完成后，需要满足以下标准：

- 1440px 下与 Figma 设计稿高度一致。
- 1028px 下保持完整桌面结构。
- 1920px 下页面舒展但不过度放大。
- 1920px 以上内容居中，仅增加两侧留白。
- 图片和视频无变形。
- 文字清晰可读。
- 组件尺寸合理。
- 内容层级明确。
- 页面间距协调。
- 不出现非预期滚动条。
- 不出现资源溢出。
- 不出现布局跳动。
- 不破坏作品集的视觉表现。
